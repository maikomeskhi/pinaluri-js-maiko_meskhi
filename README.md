### რამდენიმე კონკრეტული JS ნაწილის ახსნა (პრეზენტაციისთვის)

**1. `Promise.all()` — რამდენიმე კატეგორიის ერთდროულად წამოღება**

```js
const requests = Array.from(selectedCategories).map((cat) =>
  fetch(`${API_BASE}/filter.php?c=${cat}`).then((r) => r.json()),
);
const results = await Promise.all(requests);
```

> როცა მომხმარებელი ირჩევს რამდენიმე კატეგორიას (მაგ. "Chicken" და "Dessert"), თითოეულისთვის ცალკე fetch მოთხოვნა იგზავნება. `Promise.all` ელოდება რომ **ყველა** მოთხოვნა დასრულდეს, და მხოლოდ მერე აგრძელებს კოდს — ასე არ ველოდებით თითოეულს თანმიმდევრობით, არამედ ყველა ერთდროულად სრულდება.

**2. `Set` — არჩეული კატეგორიების შესანახად**

```js
let selectedCategories = new Set();
selectedCategories.add("Chicken");
selectedCategories.delete("Chicken");
```

> `Set` არის მასივის მსგავსი კოლექცია, მაგრამ ავტომატურად არ უშვებს დუბლიკატებს და აქვს მარტივი `.add()` / `.delete()` / `.has()` მეთოდები — იდეალურია "მონიშნულია თუ არა" ტიპის ლოგიკისთვის.

**3. Spread syntax — ობიექტის კოპირება ცვლილებით**

```js
const list = this.getShoppingList().map((item) =>
  item.id === id ? { ...item, done: !item.done } : item,
);
```

> `{ ...item, done: !item.done }` ქმნის item-ის **ახალ ასლს**, სადაც ყველა თვისება იგივეა, გარდა `done`-ისა, რომელიც ტოგლდება (true↔false). ეს არის "იმიუტაბელური" (არ ვცვლით ორიგინალს პირდაპირ) მიდგომა მონაცემების განახლებაზე.

**4. Optional chaining `?.`**

```js
document.getElementById("modal-close")?.addEventListener("click", closeModal);
```

> `?.` ამოწმებს არსებობს თუ არა ელემენტი **მანამდე** სანამ მასზე მეთოდს გამოვიძახებთ. თუ ელემენტი არ არსებობს (`null`), კოდი უბრალოდ გამოტოვებს ამ ხაზს შეცდომის მიცემის ნაცვლად.

**5. `IntersectionObserver` — სექციების და scrollspy-ის ანიმაცია**

```js
const observer = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) entry.target.classList.add("is-visible");
  });
});
```

> ეს არის ბრაუზერის ჩაშენებული ინსტრუმენტი, რომელიც "აკვირდება" ელემენტს და გვატყობინებს როცა ის ეკრანზე ჩნდება — ასე არ გვჭირდება სქროლის მუდმივი შემოწმება ხელით (`scroll` event-ის დასმენა), რაც უფრო სწრაფია წარმადობის კუთხით.

## 🌐 გამოყენებული API

[**TheMealDB**](https://www.themealdb.com/api.php) — უფასო, auth არ სჭირდება.

- `categories.php` — ყველა კატეგორია
- `filter.php?c=CategoryName` — რეცეპტები კატეგორიის მიხედვით
- `search.php?s=query` — ძებნა სახელით
- `search.php?f=letter` — ძებნა პირველი ასოთი (default სია)
- `lookup.php?i=id` — კონკრეტული რეცეპტის სრული დეტალები
